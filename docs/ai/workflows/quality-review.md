# Workflow: Quality Review

## 目的

実装結果が品質基準・アーキテクチャ方針・テスト方針を満たすかを確認する。

## 手順

1. 変更一覧の確認
2. アーキテクチャ整合性チェック
3. セキュリティ観点チェック
4. テスト網羅性チェック（正常系/境界値/異常系/回帰）
5. 保守性チェック（命名・責務・複雑度）
6. 生成コード混入チェック
7. レビュー結果の要約（重大度つき）

## 推奨アウトプット

- Critical / High / Medium / Low の区分
- すぐ直す項目 / 後続対応項目 / 監視項目

## Decision Gate

修正方針に複数案がある場合は `Decision Request` を提示し、ユーザーの選択まで停止する。
